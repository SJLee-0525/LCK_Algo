# CLAUDE.md

LCK LeetCode 스터디 레포에서 작업할 때 따르는 규칙과 자동화 지침입니다.

## 이 레포가 하는 일

Lucian · Kevin · Chloe가 LeetCode "Top Interview Questions"(Easy → Medium → Hard) 카드를 주 5문제씩 풀고, 주차 단위로 PR을 올려 서로 리뷰하는 스터디 레포.

## 멤버와 언어

| 멤버   | 파일명                       | 언어                |
| ------ | ---------------------------- | ------------------- |
| Lucian | `lucian.py` 또는 `lucian.js` | Python / JavaScript |
| Kevin  | `kevin.py`                   | Python              |
| Chloe  | `chloe.java`                 | Java                |

멤버가 추가되면 같은 규칙으로 파일만 늘리면 된다.

## 폴더 / 파일 규칙

```
week{NN}/
├── README.md                         # 그 주 문제 목록 + 진도표
└── {문제번호 4자리}-{slug}/
    └── README.md                     # 문제 설명 + 접근 포인트
```

- 주차: `week01`, `week02`, ... (2자리 0 패딩)
- 문제 폴더: `{문제번호 4자리}-{slug}` — 번호는 4자리 0 패딩(`19` → `0019`), slug은 LeetCode 제목을 소문자 + 하이픈(`Min Stack` → `min-stack`)
- 풀이 코드 파일(`lucian.py` 등)은 **각자 PR로 추가**한다. 스캐폴딩 단계에서는 만들지 않는다(빈 정답 파일 X).

## 새 주차 생성 절차 (가장 자주 하는 작업)

사용자가 "weekN 문제" 라며 LeetCode 문제 본문(제목/번호/난이도/설명/예시/제약)을 붙여넣으면:

1. `week{NN}/` 폴더를 만든다. (주차 번호는 사용자가 지정하거나, 다음 순번을 쓴다)
2. 붙여넣은 각 문제마다 `week{NN}/{번호}-{slug}/README.md` 를 아래 [문제 README 템플릿](#문제-readme-템플릿)으로 생성한다.
3. `week{NN}/README.md` 를 [주차 README 템플릿](#주차-readme-템플릿)으로 생성한다. 문제 표는 사용자가 붙여넣은 **순서대로** 나열한다.
4. 루트 `README.md`의 "진도 현황" 표에 새 주차 행을 추가한다.

### 중요 원칙

- **LeetCode를 직접 fetch하지 말 것.** `leetcode.com`은 비로그인 요청을 403으로 막는다. 문제 번호/제목/난이도/내용은 **사용자가 붙여넣은 텍스트만** 신뢰한다. 기억으로 추측해 채우지 않는다.
- 난이도는 LeetCode 표기 그대로 적는다 (Easy 카드에 있어도 본 난이도가 Medium이면 Medium).
- 접근 "포인트"는 정답 코드를 주지 말고, 방향 힌트 한두 개만 적는다 (스터디 취지).
- 기존 주차 문서의 형식과 톤을 맞춘다. 헤더에 이모지를 넣지 않는다. 굵게는 꼭 필요한 곳만.

````markdown
# {번호}. {제목} ({난이도})

- **링크**: {문제 URL}
- **주제**: {예: Stack / Design}

## 문제 요약

{붙여넣은 설명을 1~3문장으로 압축}

### 예시

```
{대표 입력 → 출력 한두 개}
```

### 제약

- {핵심 제약만}

### 포인트

- {정답이 아닌 방향 힌트 한두 줄}
````

## 주차 README 템플릿

```markdown
# Week {NN} ({시작일} ~ {종료일})

카드: [Top Interview Questions — {레벨}]({카드 URL})

{한 줄 주제 요약}

## 이번 주 문제 (5)

| #   | 문제                    | 난이도   | 주제   | 폴더              |
| --- | ----------------------- | -------- | ------ | ----------------- |
| 1   | [{번호}. {제목}]({URL}) | {난이도} | {주제} | [{번호}]({폴더}/) |
| ... |

## 진도

| 문제          | Lucian | Kevin | Chloe |
| ------------- | :----: | :---: | :---: |
| {번호}-{slug} |   ⬜   |  ⬜   |  ⬜   |
| ...           |

> ✅ 완료 · 🟡 진행중 · ⬜ 미시작
```

## 글쓰기 스타일

- 헤더에 이모지를 넣지 않는다.
- 굵게/불릿 남발 금지. 평문이 더 읽기 쉬우면 평문으로.
- 군더더기 표현 빼고 직접적으로 쓴다. (AI 티 나는 상투어 지양)

## 포맷

문서를 만들거나 고친 뒤에는 커밋 전에 `npm run format`(Prettier)을 돌린다. Markdown/JS/JSON/YAML을 정리한다. Python·Java는 Prettier 대상이 아니므로 각자 포매터에 맡긴다.
